# RMB记账 — 使用说明

## 一、项目简介

基于 Android 原生开发的短信账单自动分析工具，支持：
- 📨 自动读取手机中的银行/支付宝/微信账单短信
- 🏷️ 智能识别收入/支出，自动分类
- 📊 月度分类统计、进度条可视化
- 📅 年度总览、月度对比、月均消费
- 🔍 按月份、收入/支出筛选
- 📱 适配澎湃3.0，Material3 动态取色

---

## 二、功能说明

### 1. 明细页（首页）

- 显示所有账单记录，按时间倒序排列
- **月份选择器**：左右箭头切换月份
- **筛选按钮**：全部 / 支出 / 收入
- **汇总信息**：显示当月支出、收入、结余
- **账单列表**：每条显示分类图标、分类名、商户/银行、金额、时间
- 点击可查看详情

### 2. 统计页

- **月份选择器**：选择要查看的月份
- **总览卡片**：当月总支出和总收入
- **支出分类列表**：
  - 每个分类显示图标、名称、金额、占比百分比
  - 进度条可视化各分类占比
  - 按金额从大到小排序

### 3. 年度页

- **年份选择器**：左右箭头切换年份
- **年度总览卡片**：
  - 年度总支出、总收入、结余
  - 月均支出、月均收入
- **月度明细列表**：
  - 每月支出、收入、结余
  - 只显示有数据的月份

### 4. 设置页

- **重新读取短信**：从手机短信中重新扫描账单
- **清除所有数据**：删除所有已记录的账单数据（需二次确认）
- **使用说明**：功能介绍和使用提示

## 五、数据存储

| 数据 | 存储位置 | 说明 |
|------|----------|------|
| 账单记录 | 本地 SQLite 数据库 | 数据库名：`rmb_db` |
| 短信读取 | 系统短信 `content://sms/inbox` | 只读，不修改短信 |

- 所有数据仅存储在本地，不会上传到任何服务器
- 短信解析在本地完成，不联网
- 支持去重：相同短信内容不会重复记录

## 七、编译与安装

### 使用 Android Studio

1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. 打开项目文件夹 `rmb-app`
3. 等待 Gradle Sync 完成
4. 点击 Build → Build APK
5. 输出路径：`app/build/outputs/apk/debug/app-debug.apk`

### 使用命令行

```bash
cd rmb-app
chmod +x gradlew
./gradlew assembleDebug
```

### GitHub Actions 自动编译

1. 推送代码到 GitHub `main` 分支
2. GitHub Actions 自动开始编译
3. 编译完成后在 Actions → Artifacts 下载 APK

## 九、常见问题

**Q: 首次打开没有数据？**
A: 需要授予短信权限，授权后会自动读取。如果没有账单短信则显示"暂无数据"。

**Q: 某条账单没识别到？**
A: 短信格式不匹配解析规则。支持的格式包括银行消费通知、支付宝/微信扣款通知等。广告、验证码等非账单短信会被过滤。

**Q: 分类不准确？**
A: 分类基于短信内容中的关键词匹配。如果商户名不在关键词列表中，会归类为"其他"。

**Q: 数据会丢失吗？**
A: 数据存储在本地 SQLite 数据库中，除非手动清除或卸载应用，否则不会丢失。

**Q: 换手机数据还在吗？**
A: 数据存储在应用私有目录，换手机需要手动迁移或重新读取短信。

**Q: 支持哪些 Android 版本？**
A: 最低支持 Android 8.0（API 26），推荐 Android 12+ 以获得最佳动态取色效果。
